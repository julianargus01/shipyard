# Shipyard - Learn to Build with AI

For the person who's watched the AI tutorials, run a few AI chats, maybe bought a course — and still hasn't built anything. The one who freezes at "where do I even start."

Shipyard is a coach, not a tutorial. It won't hand you answers or build it for you — it asks the questions, pushes back when you're vague, makes you write every line yourself, and holds you to your own definition of done.

Drop the folder into a Claude or Codex project and say hello. In about 20–30 minutes you walk out with your first real, working AI tool — a small folder of files you built and actually understand.

---

## What You'll Walk Away With

In about **20–30 minutes** you will have:

- **A small, working AI tool you built yourself** — real `.md` files on your computer (your own `identity.md` + `context.md` in a folder you make).
- **An understanding of folder structure, and how it works** — the Coach teaches while you build, one step at a time.

You pick one of four practice builds: **File Organizer**, **Brain-Dump Planner**, **Notes**, or **Decision Helper**.

---

## What You Need

1. **Claude Code or Codex** — open this folder as a project (instructions below).
2. **A plain-text editor** that can save `.md` files.
   - On Mac: open TextEdit, go to **Format → Make Plain Text**, and when you save, type the filename ending in `.md` — not `.rtf`.
   - Any other plain-text editor (VS Code, Notepad, etc.) works fine.
3. **20–30 minutes.**


---

## How to Start

1. Open this `shipyard` folder as your project in Claude Code or Codex.
2. Say hello — the Coach greets you and takes it from there.

That is the whole setup; the Coach loads itself.

**If it does not greet you as your Coach** (some setups do not auto-load), paste this line to start it:

`Read identity.md and follow the startup sequence`

Each session starts fresh — just open the folder and say hello again.

---

## The Six Stages

| Stage | What happens |
|-------|-------------|
| 0 — Settle | Say hello. The Coach gets a feel for where you're starting from. |
| 1 — Pick ONE | Choose one of four preset tools. |
| 2 — Build the core files | You write your tool's two files with the Coach, one field at a time. |
| 3 — Run it | The Coach runs your tool on your real input so you can see it work. |
| 4 — Ship | Confirm it works. The Coach declares it done. |
| 5 — Show | Optional: write a short description so you can share or explain it. |

---

## File Map

These are the Shipyard system files — the Coach reads them; you don't need to touch them.

| File / Folder | What it does |
|---------------|-------------|
| `identity.md` | Loads the Coach and routes it through each stage. |
| `rules.md` | The coaching rules — how the Coach behaves, what it never skips. |
| `examples.md` | Examples of good coaching at each stage. |
| `reference/` | Per-stage detail files the Coach loads as needed. |
| `reference/voice.md` | The Coach's tone and vocabulary guide. |
| `reference/presets/` | The four preset scaffolds the Coach uses when you pick your tool. |

---

## Portable and Reusable

Everything is plain markdown. The tool you build works in any Claude Code or Codex project — just drop your project files into a new project and it runs. No installs, no accounts, no lock-in.
