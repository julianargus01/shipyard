# Phase 1 Reference

## Beginner-Friendly Project Types

| Type | What it does | Best for |
|------|-------------|----------|
| Response generator | Takes input, drafts a reply | Anyone who answers repetitive messages |
| Content repurposer | Turns one format into another | Writers, marketers, anyone with existing content |
| FAQ bot | Answers common questions about a topic | Experts who explain the same thing repeatedly |
| Summarizer | Condenses long content into key points | Anyone who reads/processes a lot of text |
| Niche post generator | Creates social content for a specific topic | Domain knowledge + audience to build |
| Simple screener | Asks questions, gives a recommendation | Consultants, coaches, anyone with a decision framework |
| Personal assistant | Handles a specific recurring personal task | Anyone with a repetitive workflow |

## Matching to the Preset
1. Match the user's pain to the most natural project type
2. Make it specific: not "a FAQ bot" — "a FAQ bot for independent landlords"
3. If none fit: "What is the most repetitive thing in your work or life right now?"

## The Specificity Rule
Generic = weak. Specific = strong.
"A response generator" → "A response generator for gym owners replying to Google reviews"

## The Parking Lot
Every idea that doesn't get picked goes here. Label it V2. Never throw ideas away — park them.
