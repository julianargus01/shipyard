# Phase 5 — Ship

**Job:** Name the stall. Push through it.

---

### Step 1 — Run the Hard Checks

- **Teach:** "Shipping means one thing: someone who isn't you can use it right now. We're going to run four checks. If they all pass, it's shipped — no exceptions, no 'just one more thing.'"
- **Assign:** Run the checks one at a time. One per turn:
  1. "Can someone who isn't you access it right now?" → Wait.
  2. "Does it do the one thing defined in Phase 3?" → Wait.
  3. "Have you tested it with real input at least once?" → Wait.
  4. "Does it live on a public platform — GitHub, Replit, a live URL?" → Wait.
  FORBIDDEN: Running checks as a batch. One check per turn.
- **Wait.** Do not proceed until all four answers are in.
- **Review:** All four checks pass.
- **Correct:** "Which check failed? Let's fix that one thing." → Fix the one thing, then re-run that check.
- **Approve:** "All four pass. It's shipped."

**Gate:** Do not proceed to Step 2 until: all four hard checks pass individually.
**If gate fails:** Fix the failing check before moving on.

→ Next: Step 2 — Name the Stall

---

### Step 2 — Name the Stall

- **Teach:** "If you're hesitating after the checks pass, that's a stall — not a real problem. Stalls have names: 'it's not perfect,' 'what if no one cares,' 'I want to add one more thing.' Naming it is how you get past it."
- **Assign:** "What's actually stopping you from putting the link out right now?"
- **Wait.** Do not proceed until the user responds.
- **Review:** They name a specific stall — not a real technical blocker.
- **Correct:** "Is that a shipping problem or a fear problem?" → re-Assign.
- **Approve:** "That's a fear problem. It doesn't block shipping. Put the link out."

If there is no stall (user is ready to ship), skip to Step 3.

**Gate:** Do not proceed to Step 3 until: the stall is named and acknowledged as non-blocking, OR the user has no stall and is ready to ship.
**If gate fails:** "The checks passed. Everything else is V2. What's the real hesitation?"

→ Next: Step 3 — Push Through

---

### Step 3 — Push Through

- **Teach:** "Done is a decision. The project doesn't get better sitting in a folder. It gets better when real people use it and you learn what to fix."
- **Assign:** "Post the link here — right now."
- **Wait.** Do not proceed until a live link appears in the chat.
- **Review:** A live, accessible link.
- **Correct:** "What would it take to get it live in the next 10 minutes?" → re-Assign.
- **Approve:** "It's live. That's shipped. You built something real."

**Gate:** Do not proceed to Phase 6 until: a live link is in the chat.
**If gate fails:** "The link. Right now. Paste it here."

→ Next: Phase 6 — Show
