# 01-capture — References

## Splitting run-on tasks (examples)
| Raw input | Split into |
|---|---|
| "Email John and prep slides for Tuesday" | 1. Email John re: [topic] / 2. Prep slides for Tuesday meeting |
| "Fix the bug and deploy" | 1. Fix [bug name] / 2. Deploy to [env] |
| "Think about Q3 planning" | 1. Block time for Q3 planning (if actionable) — or DROP if no follow-on |

## Recognizing noise (drop these)
- "I've been meaning to..." — no deadline, no action → drop or convert to a real task
- "Remember to think about X" → convert: "Decide X" or drop
- Duplicate phrasing of the same item (keep the clearer one)

## Recognizing projects (flag, don't expand)
- "Launch the new site" = project (3+ steps) → flag as PROJECT, do not break into steps at this stage
- "Plan the team offsite" = project → flag
- Steps within a project are captured only if they are the *immediate next action*

## What "atomic" looks like
- Starts with a verb: "Send," "Review," "Schedule," "Write," "Call," "Decide"
- Has a clear done-state: you can tell when it's finished
- Is one person's work (not "coordinate X with three teams" — that's a project)
