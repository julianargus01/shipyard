# Stage 02 — Organize ★

## Purpose

This is the judgment stage. Take the labeled inventory from `01-extract` and:
- Attach owners and dates to tasks where present or strongly implied
- Flag tasks that are too vague to act on
- Group key points by theme
- Deduplicate repeated items

The `★` marks this as the stage where `references.md` is applied — the user's judgment is encoded here.

## Position in flow

Second stage. Receives the labeled inventory from `01-extract`. Output feeds `03-format`.

## Why this stage matters

Raw extraction is mechanical. Organization requires judgment: what makes a task actionable? What counts as vague? How should points group? These are decisions the user encodes in `references.md`, not defaults the tool invents.
