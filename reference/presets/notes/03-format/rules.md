# Stage 03 — Rules

## Input

The organized sections from `02-organize`: Decisions, Tasks (with owner/due/flags), grouped Key Points, Open Questions.

## Process

1. Read `references.md` section 4 for the recap format and section order.
2. Write the **Summary** (2-4 sentences): what the notes were from, what mattered most. Do not introduce new facts.
3. Write the **Decisions** section: numbered list, one decision per line, past-tense declarative phrasing.
4. Write the **Action Items** table: `| Task | Owner | Due | Notes |`. Carry `[VAGUE]` flags into the Notes column verbatim.
5. Write the **Open Questions** section: bulleted list. Include all items from `02-organize`. Do not drop any.
6. Decide whether to include a **Follow-up Message Draft**: include if there are unresolved owners, open questions requiring a response, or clear next-step tasks with named owners. Omit for solo/internal notes with no audience.
7. Apply the voice from `identity.md` throughout: sentence length, tone, and level of formality.
8. Do not add sections not listed in `references.md` section 4 unless the user has added them there.

## Output

The complete clean recap, using the structure from `references.md` section 4. Markdown formatting. Ready to copy and share.

## Gate

Output is complete when:
- All four core sections are present (Summary, Decisions, Action Items, Open Questions).
- No organized items from `02-organize` were dropped or silently altered.
- `[VAGUE]` flags appear in the Action Items table exactly as carried forward.
- Voice matches `identity.md`.
- Follow-up draft is present or deliberately omitted per rule 6 above.
